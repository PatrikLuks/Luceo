### 2. Security fixes (FÁZE 1)

#### 2.1 Timing oracle v login (HIGH)
- **Problém:** Dummy hash path v login volal `_ph.verify()` přímo, zatímco reální uživatelé šli přes `verify_password()` s extra branch pro bcrypt. Různý timing odhaloval existenci emailu.
- **Fix:** Nová funkce `dummy_verify()` v `security.py` — veřejné API místo importu `_ph`. Odstraněn import `_ph` z `auth.py`.

#### 2.2 Prompt injection přes double .replace() (HIGH)
- **Problém:** `chat.py` používal `LUCEO_SYSTEM_PROMPT.replace("{rag_context}", ...).replace("{user_context}", ...)`. Pokud RAG obsah obsahoval `{user_context}`, druhý replace ho substituoval.
- **Fix:** Přechod na `string.Template` s `$` syntaxí. Nová funkce `build_system_prompt()` v `prompts.py` používá `safe_substitute()`.

#### 2.3 GDPR deletion mazala audit logy (MEDIUM → AI Act compliance)
- **Problém:** `delete_me` endpoint explicitně mazal `AuditLog` záznamy, i když FK má `ondelete="SET NULL"`. To porušovalo požadavek AI Act Art. 12 na zachování audit trail.
- **Fix:** Odstraněn řádek `delete(AuditLog).where(...)`. FK SET NULL nyní anonymizuje audit logy místo mazání.

#### 2.4 Decrypt error leak (MEDIUM)
- **Problém:** `decrypt_field()` vracela `f"Decryption failed: {e}"` — potenciální leak interních detailů.
- **Fix:** Generická zpráva `"Decryption failed"` bez interpolace.

#### 2.5 Non-atomic register (MEDIUM)
- **Problém:** Register endpoint používal 2 commity (user + audit log separátně).
- **Fix:** Nahrazeno `db.flush()` + jeden `db.commit()` na konci.

#### 2.6 Checkin upsert race condition (MEDIUM)
- **Problém:** Při concurrent requestech pro stejný den oba viděly "no checkin" a oba insertovaly → `IntegrityError`.
- **Fix:** Přidán `try/except IntegrityError` s retry (re-select + update).

#### 2.7 Decryption failure logging v chat (LOW)
- **Problém:** Decryption selhání v chat historii bylo silentně ignorováno.
- **Fix:** Přidáno `logger.warning()` pro failed decrypt.

## FÁZE 3 — Nové funkce

### 5. Health endpoint enhancement
- Přidán DB connectivity check (`SELECT 1`)
- Status: `"ok"` (DB connected) nebo `"degraded"` (DB unavailable)
- Přidáno pole `database: "connected" | "unavailable"`
- Verze zvýšena na `0.2.0`, centralizována do `APP_VERSION` konstanty

### 6. Password change endpoint (`PUT /api/v1/auth/password`)
- Response model: `MessageResponse`
- Validace: current password, min_length=8 na new password
- Odmítne same-as-current password (400)
- Automaticky revokuje všechny refresh tokeny (force re-login)
- Audit logged (`password_changed`)
- Rate limited (5/minute)

### 7. Conversation delete endpoint (`DELETE /api/v1/chat/conversations/{id}`)
- Ownership check (404 pro cizí konverzace)
- Kaskádní smazání messages
- Audit logged (`conversation_deleted`)
- Rate limited (20/minute)
- Response: 204 No Content
