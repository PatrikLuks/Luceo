# Session 7 Report — 2026-04-06

## Souhrn

Kontinuální zlepšování po Session 6. Zaměření na bezpečnostní dluhy, kvalitu API dokumentace a provozní vyspělost.

## Změny

### 1. python-jose → PyJWT migrace (CRITICAL)

- **Důvod:** python-jose je neudržovaný (poslední release 2021), má známé CVEs
- **Řešení:** Nahrazení `from jose import JWTError, jwt` za `import jwt` + `from jwt.exceptions import InvalidTokenError`
- **Dopad:** Stejné API, žádné breaking changes, pyproject.toml aktualizován
- Všechny JWT testy prošly bez úprav (kromě exception class)

### 2. OpenAPI docs skryté v produkci

- `docs_url=None` a `redoc_url=None` v produkčním prostředí
- V development režimu zůstávají dostupné na `/docs` a `/redoc`
- Integration test ověřuje dostupnost v dev mode

### 3. OpenAPI summary/description na všech endpointech

- Přidáno `summary` a `description` na všech 24 endpointech
- Jasné popisy pro frontend vývojáře i automatizované nástroje
- Ruff E501 compliance (řádky <100 znaků)

### 4. Pre-computed dummy hash

- `_DUMMY_HASH` se počítá jednou při importu modulu (ne při každém `dummy_verify()` volání)
- Úspora ~50ms na každý neúspěšný login (eliminace zbytečného argon2 hash)
- Nový test `test_dummy_hash_is_precomputed` ověřuje argon2 formát

### 5. Token cleanup endpoint

- `POST /api/v1/admin/cleanup-tokens` — mazání expired + revoked refresh tokenů
- `cleanup_expired_tokens()` v security.py — vrací počet smazaných tokenů
- Rate limited (1/minute), vyžaduje auth
- Integration test ověřuje funkčnost

### 6. pyproject.toml version fix

- Opravena nesrovnalost: `pyproject.toml` říkal `0.1.0`, `main.py` říkal `0.2.0`
- Sjednoceno na `0.2.0`
